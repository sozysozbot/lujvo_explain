<h3 id="toc_2">0章 概要</h3>

<p>ロジバンにおいて、lujvo（複合語）を作る規則は意外とめんどくさいし、聖典CLLの説明は分かりにくい。さらに、英語資料は一応あるもののまとまった日本語資料が見当たらない。<br>
ということで、jvozbaを再発明 (URL: https://goo.gl/hchjGj) した筆者が実装時のわーきゃーを踏まえながら解説記事を書いてみようと思う。<br>
なお、筆者が理解していないので、意味論には触れない。  </p>

<hr>

<h3 id="toc_3">1章 rafsi</h3>

<p>「CLL 4.6に丸投げします」と、言えたらどんなに嬉しいことか。<br>
困ったことに、CLLはrafsiに関して自己矛盾を含んでいるのである。<br>
その話は第6章で触れる<sup><a href="#foot1">1</a></sup>として、ここではサラッとrafsiについて解説するのみにしておく。<br>
rafsiとは、複合語(lujvo)を生み出すための形態素で、その形には8種類ある。
Cが子音、CCが「語頭に立ちうる二重子音」を表す。C/Cは「語中にあり得る二重子音」。</p>

<p>結論から言えば、lujvoはこのrafsiをくっつけていけば作れるのだが、そのくっつけるための規則がめんどくさいのである。以下その規則を解説していく。</p>

<hr>

<h3 id="toc_4">2章 rafsiをくっつけるPart1</h3>

<p>まず大前提として、lujvoはbrivlaなので、lujvo末には母音で終わるrafsiが来なくてはならない。<br>
それが済んだらrafsiをくっつけていけばいいが、諸事情により「ハイフン字」（<code>y</code>, <code>n</code>, <code>r</code>）がrafsi間に挿入されることがある。「諸事情」とは、</p>

<ul>
<li>4文字rafsi<br></li>
<li>禁則回避<br></li>
<li>剥がれ落ち防止<br></li>
</ul>

<p>の3パターンである。まずは前2つを解説する。</p>

<h4 id="toc_5">2-1 4文字rafsi</h4>

<p>4文字rafsiの後ろには必ず<code>y</code>が必要である。  </p>

<p>例：<br>
-plis- + -ladru- → plis<strong>y</strong>ladru</p>

<p>これは、lujvoを一意に分解するのに必要な規則である。lujvoの2文字目には絶対にyが来ないことが知られているので、そのこととこの規則から、先頭を読むだけで {plisyladru} が *[-pli- + syladru] ではなく [-plisy- + ladru] であることを知ることができるのだ。流石LLG、賢い。<br>
ちなみにこの例でyを抜かすと{plisladru}となるが、これは[-pli- + -sla- + -dru-]である。CVC/C型については{karctadji}を考えてみよう。</p>

<h4 id="toc_6">2-2 禁則回避</h4>

<p>そのままくっつけると禁則な子音列が発生する場合、2つのrafsiの間には<code>y</code>が要求される。  </p>

<p>例：<br>
-fam- + -ma&#39;o- → fam<strong>y</strong>ma&#39;o （同一子音の連続を回避）<br>
-fas- + -bau- → fas<strong>y</strong>bau （無声阻害音＋有声阻害音を回避）<br>
-cen- + -dje- → cen<strong>y</strong>dje （<code>ndj</code>を回避）  </p>

<p>この規則はロジバンの音韻構造の制約を直に受けているだけのものであり、それ以上に深い意味は特にない。</p>

<hr>

<h3 id="toc_7">3章 rafsiをくっつけるPart2</h3>

<p>さて、「lujvoを作る」というのは、そもそもどういうことだろうか。 <sup><a href="#foot3">3</a></sup><br>
私の考えでは、「ハンバーガー🍔をつくること」に例えられると思う。<sup><a href="#foot4">4</a></sup><br>
ハンバーガーの内側の具を入れ替えても、それはまだハンバーガーである。 <sup><a href="#foot5">5</a></sup><br>
しかし、「一番上に丸くてゴマのついたパン、一番下に平らなパン」という掟を破って層を配置すれば、ハンバーガーではない。  </p>

<p>lujvoにおいても同様のことが言える。<br>
前述のとおり、lujvo末のrafsiは母音終わりであることが要求されるが、lujvoの語頭に来るrafsiにも面倒な規則が存在するのだ。  </p>

<p><strong>☆以下の操作は、具を載せ終わって、最後に上の丸いパンを載せるときにのみ、一度だけ、行うこと☆</strong> （もちろん、具や丸いパンを載せていく過程で2-1と2-2の操作は行うこと） </p>

<ul>
<li>case 1: 丸いパンが4文字rafsiまたはCCV rafsiの場合<br>

<ul>
<li>特別な操作は不要<sup><a href="#foot6">6</a></sup></li>
</ul></li>
<li>case 2: [CVV + CCV]で単語が完成する場合（例： {ma&#39;ovla}, {fu&#39;ivla}）

<ul>
<li>特別な操作は不要</li>
</ul></li>
<li>case 3: 丸いパンがCVVで、case 2以外のとき

<ul>
<li>CVVの後に<code>r</code>を挿入（例： {fa&#39;o<strong>r</strong>ma&#39;o}, {fu&#39;i<strong>r</strong>vlarafsi}）</li>
<li>ただし、直後に既に<code>r</code>がある場合は挿入されるのは<code>n</code>（例： {ka&#39;e<strong>n</strong>ri&#39;u}）</li>
</ul></li>
<li>case 4: 丸いパンがCVCのとき

<ul>
<li>2-2が適用されるなら普段通り。さもなくば3-1へ進む</li>
</ul></li>
</ul>

<h4 id="toc_8">3-1 tosmabruテスト</h4>

<p>先頭にCVCを付けたものがαとβのどちらか片方に該当する場合にのみ、α, βの形ではなく、先頭のCVCの後に<code>y</code>をねじ込んだ形が採用される。ちなみに、これをtosmabruテストと呼ぶ。   </p>

<p>α [（1個以上のCVC） + CVCCV<sup><a href="#foot7">7</a></sup>] かつ、くっつけた時全ての子音結合がCCである<br>
β [（2個以上のCVC） + <code>y</code> + （1個以上の任意のrafsi）] かつ、y以前の全ての子音結合がCCである  </p>

<p>従って、</p>

<ul>
<li>[-cas- -tadji-] → {cas<strong>y</strong>tadji} （全てCC）</li>
<li>[-kan- -tadji-] → {kantadji} （<code>nt</code>はC/C）</li>
<li>[-cas- -kan- -tadji-] → {caskantadji} （<code>sk</code>はCCだけど<code>nt</code>はC/C）</li>
<li>[-pas- -paz- -badna-] → {paspazbadna} （<code>dn</code>がC/C）</li>
<li>[-pas- -pas- -y- -badna-] → {pas<strong>y</strong>pasybadna} （<code>dn</code>はC/Cだが判定にかかるのは<code>paspasy</code>の部分まで）</li>
</ul>

<h3 id="toc_9">4章 一意性の証明</h3>

<p><strong>【長文注意】</strong>  </p>

<p>以下、上記の方法でlujvoを作れば「(0個以上のcmavo) + lujvo」が「(0個以上のcmavo) + 2個以上のrafsiの列」に一意に分解できることの構成的な証明を行う。なお、以下、C\Cで「語頭に立てない二重子音」を表すとする。  </p>

<h4 id="toc_10">4-1 lujvoの先頭部分に関する考察</h4>

<p>まず、3章の全パターンを列挙して調べることで、lujvoの先頭部分としてありうる形が限られることが分かる。（2番目のrafsiが何であれ、必ずCで始まり、その次がyでないことは保障されていることに注意）<br>
便宜上、それぞれの形に番号を振り、先頭のrafsiを強調表示する。</p>

<table>
<thead>
<tr>
<th>パターン</th>
<th>形</th>
</tr>
</thead>

<tbody>
<tr>
<td>case1</td>
<td>① <strong>CCVC</strong>yC-<br>② <strong>CVC/C</strong>yC-<br>③ <strong>CCV</strong>C(y以外の文字)-</td>
</tr>
<tr>
<td>case2</td>
<td>④ <strong>CVV</strong>CCV</td>
</tr>
<tr>
<td>case3</td>
<td>⑤ <strong>CVV</strong>rC(y以外の文字)-<br>⑥ <strong>CVV</strong>nr(y以外の文字)-</td>
</tr>
<tr>
<td>case4 (2-2が適用されるとき)</td>
<td>⑦ <strong>CVC</strong>yC(y以外の文字)-</td>
</tr>
<tr>
<td>case4 (3-1が適用されるとき)</td>
<td>⑧ <strong>CVC</strong>yCVCCVC..CVCCV<br>⑨ <strong>CVC</strong>yCVC..CVCy-</td>
</tr>

<p>これにより、まず、2-1で書いた主張「lujvoの2文字目には絶対にyが来ない」が示され、また「lujvoの3文字目にも絶対にyが来ない」ことも分かる。また、異なる第一rafsiに由来するlujvoの形が重複することがなく、lujvoの形から一意に第一rafsiが判定できることがわかる。<br>
さらに、⑤⑥⑧⑨では第一rafsiの後ろにハイフンが挿入されているが、このハイフンが仮に存在しなかったとしても他種のrafsi列と「は」混同することがない。<sup><a href="#foot8">8</a></sup>ゆえに、第一rafsiを取り除いた後の文字列について同様の議論を行うことで<sup><a href="#foot9">9</a></sup>、lujvoを構成するrafsi列は一意に決定できる。</p>

<h4 id="toc_11">4-2 cmavo: 0個 vs. 1個以上</h4>

<p>次に、「lujvo」が「(1個以上のcmavo) + lujvo」と一致することはないことを示す。そのために、一致する例が存在するとして矛盾を導く。<br>
一致する例が存在するならば、4-1より、「(1個以上のcmavo) + lujvo」と解釈した時のcmavoの形としてありうるのは1つのCV（②⑦⑧⑨⑩）か1つのCVV（④⑤⑥）である。しかし、②④⑤⑥⑦⑧⑨⑩のいずれの場合でも、「cmavo + lujvo」との解釈は成り立たないことを示す。<br>
②④⑤⑥⑦⑧⑨の場合は単純である。</p>

<table>
<thead>
<tr>
<th>パターン</th>
<th>理由</th>
</tr>
</thead>

<tbody>
<tr>
<td>②</td>
<td>CVを取り除くと3文字目がyで、これが4-1の結論と矛盾</td>
</tr>
<tr>
<td>④</td>
<td>CVVCCVのCVVの内部にはアクセントがあるので、cmavoにはなれない</td>
</tr>
<tr>
<td>⑤⑥</td>
<td>CVVを取り除くとlujvoはrCまたはnrで始まることとなるが、これはどちらもCCではない</td>
</tr>
<tr>
<td>⑦⑧⑨</td>
<td>CVを取り除くと2文字目がyで、これが4-1の結論と矛盾</td>
</tr>
</tbody>
</table>

<p>さて、問題は⑩である。⑩の先頭からCVを取り除いた形がlujvoとなる可能性を否定しなければならない。<br>
⑩は、3-1のtosmabruテストに該当しない場合なので、</p>

<ol>
<li>[（1個以上のCVC） + CVCCV] であるが、くっつけた時一部の子音結合がC\Cである</li>
<li>[（2個以上のCVC） + <code>y</code> + （1個以上の任意のrafsi）] であるが、y以前の一部の子音結合がC\Cである </li>
<li><code>y</code>を含まず、[（1個以上のCVC） + CVCCV]でもない</li>
<li><code>y</code>を含んでいるが、[（2個以上のCVC） + <code>y</code> + （1個以上の任意のrafsi）] ではない</li>
</ol>

<p>という4通りの場合があり得る。  </p>

<p>まず1.の場合、先頭からCVを取り除くとCCVCCV..C\CV..CVCCVといった形になる。左から1つずつrafsiを削っていくとき、4-1.よりその削り方は一意に決定できる（CCVという形で必ず削られる）が、すると途中でC\CV-という形に遭遇してしまう。当然これはrafsiとして許されないので、こう分解することは許容されない。2.も同様であり、CCVCCV..C\CV..CCVCy-という形を左から削っていくとC\CV-に遭遇してしまう。</p>

<p>3.の場合、元々の分解方法では途中で「CVCCVでも4文字rafsiでもCVCでもないrafsi」に遭遇することとなる。そのrafsiの左には1つ以上のCVC rafsiがあるので、lujvoの先頭からCVを削った場合</p>

<ul>
<li>⑪ <strong>CVC</strong>/CVV- [CVVが後続する時]</li>
<li>⑫ <strong>CVC</strong>/CCV- [CCV, CCVC, CCVCVが後続する時]</li>
<li>⑬ <strong>CVC</strong>/CVC\CV [CVC\CVが後続する時]</li>
</ul>

<p>から先頭のCVが（直接的に、またはCCVCCV..CCVの最後2文字として吸収されて）失われた場合にrafsi列として成立しないことを示せばよい。
まず、上での3文字目と4文字目の2つのCがC\Cである場合、先頭のCVを失うとrafsi列と見ることはできない。次に、CCVV-, CCCV-で始まるrafsi列はないので⑪⑫は除外でき、CCVC\CVという形はCCVを削るしかなくC\CVが露出し不適。以上より3.も否定できる。</p>

<p>最後に4.の場合。2.の場合を除いていることから、[CVC + (CVCが0個以上) + (CVC以外が1個以上) + (CVCが0個以上) + <code>y</code> + （1個以上の任意のrafsi）] として表せる。つまり、「CVC+CVV」「CVC+CCV」「CVC+CCVC」「CVC+CVC/C」のいずれかの組み合わせが列中に存在するが、最初の3つは先頭のCVが直接的または間接的に削られた際CCVV-またはCCCV-というrafsi列としてあり得ないものとなり、「CVC+CVC/C」は2-1より「CVC+CVC/C+<code>y</code>」であり、先頭のCVが消えるとCCV + C/Cy-というあり得ない形となる。</p>

<p>以上より、「lujvo」が「(1個以上のcmavo) + lujvo」と一致することはないことが示された。</p>

<h4 id="toc_12">4-3 最後に</h4>

<p>最後に、2種類の「(0個以上のcmavo) + lujvo」が一致することはないことを示す。「(m個以上のcmavo) + lujvo1」からなるAと、「(n個以上のcmavo) + lujvo2」Bから導き出される文字列が一致したと仮定して矛盾を導く。<br>
まず、m=nの場合を考える。この場合、明らかにcmavo列は一致していて、ゆえにlujvo1とlujvo2も一致する。次にm&gt;nのとき、A,Bの先頭から共にn個のcmavoを削ると「((m-n)個のcmavo) + lujvo1」と「lujvo2」が一致することになるが、これは4-2に反する。AとBを入れ替えればm&lt;nの場合も示せる。</p>

<p>以上より、一意性が示された。</p>

<h3 id="toc_13">5章 おまけ1: lujvoの得点の話</h3>

<p>lujvoには「得点」が割り振られている。同じrafsi列から作られたlujvoの中でも、得点が最も低いものが「好ましい」（原文: preferable）とされている。<br>
得点の計算式自体はCLL4.12に載っているが、厳密に1の位まで得点を計算する必要があることはほとんどなく、以下の目安を覚えておけば十分であろう。  </p>

<ol>
<li>短いlujvoが勝ち（ただし、アポストロフィは0.5字相当）<br></li>
<li>引き分けならハイフン字（<code>y</code>, <code>n</code>, <code>r</code>）が少ない方が勝ち<br></li>
<li>それでも引き分けならCVC &lt; CCV &lt; Cvv<br></li>
</ol>

<h3 id="toc_14">6章 おまけ2: CLLの自己矛盾</h3>

<p>公式の説明CLL4.6は自己矛盾を含む。というのも、</p>

<blockquote>
<p>Each gismu always has at least two rafsi forms; one is the gismu itself (used only at the end of a lujvo), and one is the gismu without its final vowel (used only at the beginning or middle of a lujvo).</p>
</blockquote>

<p>という記述ををbroda, brode, brodi, brodo, broduに当てはめれば、-brod-はこの5種のgismuを表すrafsiになるはずだが、そうするとその直前にある</p>

<blockquote>
<p>each such rafsi represents only one gismu.</p>
</blockquote>

<p>に矛盾するからだ。<br>
ちなみに、CLL4.15を見ると、</p>

<blockquote>
<p>The following gismu were not made by the gismu creation algorithm. </p>
</blockquote>

<p>という文章の後にbrodVが紹介されているので、brodVが公式見解上gismuであることは確実である。<br>
詳しくは、https://goo.gl/zM7M9u を参照のこと。</p>
